反选确认弹窗（新增）

点击"反选"按钮后弹出 · 说明反选逻辑
反选确认

反选将：

✓ 取消当前已勾选的 3 个商品

✓ 勾选剩余未选择的 3 个商品

⚠️ 已发货的商品不会被影响

反选逻辑说明
· 反选 = 取消当前已勾选的，选择还没勾选的
· 只对当前筛选结果有效（如果筛选"未打印"，只取消未打印中已勾选的）
· 已发货商品不受影响

改进前后对比

问题 vs 解决方案
❌ 改进前（问题）
打印标签勾选一次 → 打单发货再勾选一次（重复操作）
复选框太小（16×16px），点击困难，仓库人员抱怨
只能看"全部"，无法筛选已打印/未打印
没有反选功能，用户不知道怎么批量操作
发货后想继续处理，不知道去哪找
✓ 改进后（优化）
打印标签勾选 → 自动同步到打单发货（一次搞定）
复选框放大到 24×24px + 整行可点击 + 行高48px
Tab筛选（全部/未打印/已打印），未打印默认在前
反选功能 + 确认弹窗说明逻辑
发货成功弹窗提供"继续处理"跳转按钮
状态同步机制（技术实现）
1. 新增字段 order_item.print_status（0=待勾选 1=已勾选待发货 2=已发货）
2. 打印标签弹窗点击"确认打印" → API标记 print_status=1
3. 打单发货页面读取 print_status=1 的商品并显示勾选
4. 打单发货点击"确认发货" → API标记 print_status=2
5. 刷新页面状态保持，不会丢失
