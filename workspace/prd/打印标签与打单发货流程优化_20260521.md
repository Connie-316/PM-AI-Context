## 1. 背景与目标

### 1.1 背景

当前仓库人员在商家端的发货流程存在严重效率问题：

```
现状流程（低效）：
打印标签弹窗 → 勾选商品（勾一次）
    ↓
打单发货页面 → 再次勾选商品（重复勾第二次）
    ↓
发货记录 → 手动找回去继续处理
```

**问题总结：**
1. 打印标签 和 打单发货 需要重复勾选相同商品，浪费时间
2. 全选/勾选按钮热区太小（点击困难，仓库人员操作很累）
3. 已打印/未打印状态无法筛选，只能看全部
4. 缺少反选功能，用户不知道如何操作
5. 打单发货后想继续处理，没有跳转入口

### 1.2 目标

```
1. 打印标签勾选 → 自动同步到打单发货（不需要重复勾选）
2. 按钮热区优化（扩大点击区域，降低误触）
3. 打印标签列表支持按"已打印/未打印/全部"筛选
4. 支持反选功能，并明确告知用户逻辑
5. 打单发货后支持"继续处理"跳转
```

### 1.3 上线后影响

- 仓库人员操作时间预计减少 50%（减少重复勾选）
- 操作错误率降低（状态清晰，不容易漏勾）
- 仓库人员满意度提升

## 3. 功能需求

### 3.1 同步机制：打印标签 → 打单发货

#### 3.1.1 核心逻辑

```
打印标签弹窗勾选商品 → 点击"确认打印"
    ↓
系统标记这些商品为"已勾选待发货"状态（不是"已打印"）
    ↓
打单发货页面自动同步这些商品（已勾选状态）
    ↓
仓库人员只需确认+点击发货 → 状态变为"已发货"
```

**关键：区分"已勾选待发货"和"已打印标签"两个状态**

| 状态 | 含义 | 在哪里显示 |
|------|------|---------|
| 待勾选 | 还没操作 | 打印标签弹窗 |
| 已勾选待发货 | 打印标签弹窗勾选了，还没发货 | 打单发货页面 |
| 已发货 | 打单发货完成了 | 发货记录 |

#### 3.1.2 同步规则

```
规则1：打印标签弹窗勾选后，打单发货页面自动显示勾选状态
规则2：打印标签弹窗取消勾选，打单发货页面同步取消
规则3：打单发货页面也可单独勾选，不影响打印标签弹窗状态
规则4：打单发货"确认发货"后，状态变为"已发货"，打印标签弹窗不再显示
```

#### 3.1.3 数据同步技术实现

```
方案A：前端本地存储（localStorage）
- 优点：实时、无需后端改造
- 缺点：换浏览器/清除缓存会丢失

方案B：后端状态标记
- 优点：持久化、不丢失
- 缺点：需要后端配合改造

推荐方案B（后端标记）：
1. 新增字段：order_item.print_status（0=待勾选 1=已勾选待发货 2=已发货）
2. 打印标签弹窗勾选 → 调用API标记 print_status=1
3. 打单发货页面读取 print_status=1 的商品并显示勾选
4. 打单发货确认 → 标记 print_status=2
```

### 3.2 按钮热区优化

#### 3.2.1 当前问题

```
当前：复选框太小（16×16px），仓库人员手指/鼠标点击困难
影响：误触多、效率低、员工抱怨
```

#### 3.2.2 优化方案

```
优化1：复选框放大到 24×24px（最小）
优化2：整行可点击（点击行任意位置toggle勾选）
优化3：全选按钮扩大热区（不只是文字，是整个按钮区域）
优化4：按钮padding增加（更容易命中）
```

**按钮尺寸规范：**

| 元素 | 当前尺寸 | 优化后尺寸 |
|------|---------|---------|
| 复选框 | 16×16px | 24×24px |
| 全选行高 | 40px | 48px（全行可点击） |
| 操作按钮padding | 8px 12px | 12px 20px |

### 3.3 打印标签列表筛选功能

#### 3.3.1 筛选维度

```
Tab切换：全部 / 未打印 / 已打印

| Tab | 显示内容 |
|-----|---------|
| 全部 | 所有商品（含已打印、未打印） |
| 未打印 | print_status=0或1的商品（还没发货的） |
| 已打印 | print_status=2的商品（已发货的） |
```

#### 3.3.2 排序规则

```
未打印的商品默认排在前面（方便仓库人员优先处理）
已打印的商品按时间倒序排列
```

#### 3.3.3 视觉区分

```
未打印商品：正常显示，白底黑字
已打印商品：灰色背景 + 删除线 + "已发货"标签
当前勾选（待发货）：蓝色背景 + 勾选状态
```

### 3.4 反选功能

#### 3.4.1 反选逻辑说明

**反选 = 取消当前已勾选的，选择还没勾选的**

```
当前勾选了：商品A、商品B、商品C（总共3个）
反选操作后：商品D、商品E、商品F（剩下3个被选中）

说明：反选只对当前筛选结果有效
- 如果筛选"未打印"，反选只取消"未打印"中已勾选的
- 已发货的商品不受影响
```

#### 3.4.2 用户提示（Modal确认）

```
操作反选前，弹出提示：

┌────────────────────────────────────────┐
│  反选确认                              │
│                                        │
│  反选将：                               │
│  ✓ 取消当前已勾选的 [3] 个商品          │
│  ✓ 勾选剩余未选择的 [3] 个商品          │
│                                        │
│  ⚠️ 已发货的商品不会被影响              │
│                                        │
│        [取消]          [确认反选]      │
└────────────────────────────────────────┘
```

### 3.5 打单发货后继续处理跳转

#### 3.5.1 交互流程

```
打单发货确认 → 弹出成功提示（含跳转按钮）
    ↓
┌────────────────────────────────────────┐
│  ✓ 发货成功                             │
│                                        │
│    已发货 [X] 个商品                    │
│                                        │
│  [继续处理]  [查看发货记录]  [关闭]    │
│                                        │
└────────────────────────────────────────┘

继续处理 → 跳转打单发货列表，从待发货状态继续
查看发货记录 → 跳转发货记录页面
关闭 → 关闭弹窗
```

#### 3.5.2 跳转参数

```
跳转打单发货列表时，自动筛选：
- 状态：待发货（print_status=1）
- 订单号：[当前订单号]
- 显示刚才发货的商品（高亮）
```

## 4. 核心流程

### 4.1 优化后的发货流程

```
Step 1：仓库人员在"打印标签弹窗"勾选本次要发货的商品
        ↓
Step 2：点击"确认打印" → 系统标记为"已勾选待发货"
        ↓
Step 3：进入"打单发货"页面 → 商品已自动勾选（同步）
        ↓
Step 4：确认无误 → 点击"确认发货"
        ↓
Step 5：弹出成功提示 → 点击"继续处理"
        ↓
Step 6：跳转到打单发货列表 → 继续处理下一个订单
```

### 4.2 打印标签弹窗流程

```
打开打印标签弹窗
    ↓
看到筛选Tab：全部 / 未打印 / 已打印（默认"未打印"）
    ↓
勾选本次要发货的商品（全选 / 单个 / 反选）
    ↓
点击"确认打印"
    ↓
系统标记 print_status=1（已勾选待发货）
    ↓
弹窗关闭，打单发货页面可同步看到勾选
```

### 4.3 打单发货页面流程

```
打开打单发货页面
    ↓
自动显示"已勾选待发货"的商品（从打印标签弹窗同步）
    ↓
可额外勾选/取消勾选（不影响打印标签弹窗状态）
    ↓
确认无误 → 点击"确认发货"
    ↓
标记 print_status=2（已发货）
    ↓
弹出成功提示 → 可跳转继续处理
```

## 6. 数据结构

### 6.1 新增/修改字段

```sql
-- 订单商品表新增字段
ALTER TABLE order_item ADD COLUMN print_status TINYINT DEFAULT 0 COMMENT '打印状态：0=待勾选 1=已勾选待发货 2=已发货';

-- API接口设计

-- 1. 标记商品为已勾选待发货（打印标签弹窗 → 确认打印）
POST /api/order/item/mark-printed
Request: { order_id: 123, item_ids: [1,2,3] }
Response: { code: 0, message: "success" }

-- 2. 获取订单商品列表（含print_status）
GET /api/order/item/list?order_id=123
Response: {
  data: [
    { id: 1, name: "模具配件A", print_status: 1, ... },
    { id: 2, name: "标准件B", print_status: 0, ... }
  ]
}

-- 3. 确认发货（打单发货 → 确认发货）
POST /api/order/item/confirm-ship
Request: { order_id: 123, item_ids: [1,2,3], logistics: "顺丰", waybill_no: "SF123" }
Response: { code: 0, message: "success" }
```

## 9. 验收标准

### 9.1 同步功能

- [ ] 打印标签弹窗勾选商品后，打单发货页面自动显示勾选
- [ ] 打印标签弹窗取消勾选，打单发货页面同步取消
- [ ] 打单发货确认发货后，商品不再显示在待发货列表
- [ ] 刷新页面后状态保持（不丢失）

### 9.2 按钮热区

- [ ] 复选框尺寸不小于 24×24px
- [ ] 点击整行（不是只点击复选框）可以toggle勾选
- [ ] 全选按钮热区扩大，整行可点击
- [ ] "确认打印"/"确认发货"按钮高度不小于 48px

### 9.3 筛选功能

- [ ] 打印标签弹窗有"全部/未打印/已打印" Tab切换
- [ ] 默认显示"未打印"
- [ ] 未打印商品排在前面
- [ ] 已打印商品有灰色背景+删除线+标签

### 9.4 反选功能

- [ ] 有反选按钮
- [ ] 反选前有确认弹窗，说明反选逻辑
- [ ] 反选只影响当前筛选结果，不影响已发货商品

### 9.5 跳转功能

- [ ] 发货成功后弹窗有"继续处理"按钮
- [ ] 点击"继续处理"跳转到打单发货列表（筛选待发货状态）
- [ ] 点击"查看发货记录"跳转到发货记录页面
