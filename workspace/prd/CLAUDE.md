## 生命周期

```
drafts/ 想法 → workspace/prd/ 撰写中 → 已完成 → 已上线 → context/02_功能模块/ 已归档
```

### 晋升规则

| 阶段 | 触发条件 | 操作 |
|------|---------|------|
| 进行中 → 已完成 | PRD 写完 + 评审通过 + 研发确认签字 | 更新 _PRD状态追踪.md 状态 |
| 已完成 → 已上线 | 功能实际上线生产环境 | 更新 _PRD状态追踪.md，发送归档提醒 |
| 已上线 → 归档 | PM 调用 doc-archive Skill | 提取功能描述到 context/02_功能模块/ |

### 自动晋升提醒

当检测到以下情况时，自动提醒晋升：
- PRD 写完+研发已确认 → 提醒"可以进入开发阶段了"
- 迭代规划中标记为"已上线" → 提醒"要不要归档到 context？"
- PRD 在 workspace/prd/ 超过 30 天没动 → 提醒"状态是否更新？"
