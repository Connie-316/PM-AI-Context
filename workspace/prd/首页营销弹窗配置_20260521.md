## 1. 背景与目标

### 1.1 背景
- 平台目前没有可配置的首页弹窗功能，营销活动只能通过开发发版上线，响应速度慢
- 不同端（PC / 小程序）图片尺寸规范不统一，运营每次配置需要手动换算
- 弹窗显示逻辑不完善，用户无法手动关闭，且重复弹出会影响用户体验
- 单个弹窗无法满足多营销活动并行展示需求

### 1.2 上线后影响
- 运营可自主配置弹窗，营销活动从"发版上线"变为"配置即上线"
- 减少运营与研发的沟通成本，提升活动响应速度

### 1.3 目标
- 运营可在平台端自主配置 PC 端和小程序端的营销弹窗，无需发版
- 支持轮播模式，一个弹窗位置可展示多个营销banner轮播
- 统一不同端的图片尺寸规范，减少运营沟通成本
- 用户可手动关闭弹窗，且七天内不再自动展示

## 3. 功能需求

### 3.1 弹窗配置管理（平台端）

#### 3.1.1 新建 / 编辑弹窗

| 字段 | 类型 | 说明 |
|------|------|------|
| 弹窗名称 | 文本 | 仅供运营识别，如"618大促banner" |
| 端类型 | 单选 | PC端 / 小程序端 |
| 展示模式 | 单选 | 单图模式 / 轮播模式 |
| 弹窗图片 | 图片 | 单图模式下上传一张；尺寸限制见3.1.2 |
| 轮播图片列表 | 图片组 | 轮播模式下传2-6张；尺寸限制见3.1.2 |
| 活动开始时间 | 时间 | 弹窗整体可见的开始时间 |
| 活动结束时间 | 时间 | 弹窗整体可见的结束时间 |
| 图片开始时间 | 时间 | 单图模式下该图片的展示时间窗口 |
| 图片结束时间 | 时间 | 单图模式下该图片的展示时间窗口 |
| 点击后跳转页面 | 外链 | 跳转目标页面类型，见3.1.4 |
| 手动关闭 | 开关 | 开启：用户可关闭；关闭：弹窗右上角不显示×，用户无法关闭 |
| 不再显示周期 | 数字 | 关闭后几天内不再显示，默认7天 |
| 状态 | 开关 | 启用 / 禁用 |

#### 3.1.2 图片尺寸规范

| 端 | 尺寸（宽×高 px） | 推荐比例 |
|----|-----------------|---------|
| PC端 | 600 × 450 | 4:3 |
| 小程序端 | 750 × 750 | 1:1（方形） |

> 尺寸不符时，系统提示"图片尺寸不符合[端]规范要求"，不允许上传

#### 3.1.3 轮播模式规则

- 轮播图片数量：2-6张
- 每张图片可单独设置「图片开始时间」和「图片结束时间」，与主配置解耦
- 每张图片可单独设置跳转页面（跳转参数独立）
- 轮播支持手动滑动（用户左右滑动切换图片）
- 轮播底部显示指示点：**1张图时不显示，2张及以上显示**
- 轮播自动切换时间固定为3秒

#### 3.1.4 跳转页面类型

| 页面类型 | 跳转路径示例 |
|---------|-------------|
| 商品列表页 | /pages/goods/list?category_id=xxx |
| 海报页 | /pages/poster/view?id=xxx |
| 商品详情页 | /pages/goods/detail?goods_id=xxx |
| 商品分组 | /pages/goods/group?group_id=xxx |
| 不跳转 | 空（纯图片展示，用户点击图片不触发任何跳转） |

### 3.2 时间规则

#### 3.2.1 两层时间概念

| 时间层 | 字段 | 作用 |
|--------|------|------|
| 活动整体时间 | 活动开始 / 结束时间 | 弹窗可见的总窗口，超过则整个弹窗不展示 |
| 图片展示时间 | 图片开始 / 结束时间 | 该图片在轮播中展示的时间窗口 |

> 单图模式下，图片展示时间必须落在活动整体时间范围内；轮播模式下，每张图片各自的时间也必须落在活动整体时间范围内。

#### 3.2.2 时间约束规则

**约束1：图片时间 ≤ 活动整体时间**
- 图片开始时间 ≥ 活动开始时间
- 图片结束时间 ≤ 活动结束时间
- 不满足时提示运营"图片展示时间不能超出活动整体时间范围"

**约束2：图片时间可重叠、可断续**
- 不要求首尾相连
- 允许同一时间段有多张图片（如"周年庆"和"618"同时段各一张）
- 允许某张图片先下线

**约束3：展示时的动态过滤**
- 当前时间不在某张图片的时间窗口内 → 该张图片跳过
- 全部图片都被跳过 → 弹窗不展示，不写关闭记录

#### 3.2.3 举例说明

> 活动整体：2026-06-01 ~ 2026-06-30
>
> - 图片A：6.01~6.10 → 6.12访问时展示
> - 图片B：6.05~6.15 → 6.12访问时展示
> - 图片C：6.20~6.30 → 6.12访问时跳过
>
> 用户6.12访问 → 只看到A和B轮播，C不出现
> 用户7.01访问 → 活动已结束，弹窗不展示

### 3.3 弹窗展示规则

#### 3.3.1 多弹窗优先级

当存在多条启用且在有效期内的弹窗配置时：

| 优先级 | 条件 |
|--------|------|
| 第1优先 | 按创建时间倒序，取第1条 |
| 后续忽略 | 同一端同时只展示1个弹窗，其他不展示 |

#### 3.3.2 触发条件（需同时满足）

- 用户打开小程序 / PC站
- 当前时间在「活动整体时间」内
- 当前时间至少有一张图片在「图片展示时间」内
- 弹窗状态为"启用"
- 该弹窗未被用户在N天内关闭过

#### 3.3.3 关闭交互规则

| 手动关闭开关 | 右上角× | 点击结果 |
|-------------|---------|---------|
| 开启 | 显示 | 点击后弹窗消失，写入关闭记录 |
| 关闭 | 不显示 | 用户无法关闭，弹窗只能等活动结束后自动消失 |

#### 3.3.4 异常情况

| 场景 | 处理方式 |
|------|---------|
| 活动整体时间外 | 弹窗不展示，不写关闭记录 |
| 所有图片都在非展示时间段 | 弹窗不展示，不写关闭记录 |
| 单张轮播图片加载失败 | 跳过该张，轮播继续；全部失败则不展示弹窗 |
| 点击的跳转页面已被删除 | 弹窗无反应，静默失败 |
| 重复点击关闭按钮 | 只执行一次，不重复写记录 |
| 轮播只有1张图 | 降级为单图模式，不显示指示点 |
| PC和小程序两端 | 关闭记录各自独立，互不影响 |

## 4. 核心流程

### 4.1 运营配置流程

```
运营在平台端新增弹窗
    ↓
填写端类型、弹窗名称、选择展示模式
    ↓
设置活动整体时间（开始 / 结束）
    ↓
上传图片并为图片设置展示时间
    ↓
配置跳转页面、关闭规则
    ↓
提交并启用
    ↓
立即生效（无延迟）
```

### 4.2 用户端展示流程

```
用户打开小程序 / PC站
    ↓
查找「启用 + 当前时间在活动整体时间内」的配置
    ↓
有多条时按创建时间倒序取第1条
    ↓
检查该弹窗是否有至少一张图片在当前时间段内？
    ↓ 否 → 弹窗不展示，结束
    ↓ 是 → 继续
    ↓
检查本地关闭记录（该弹窗ID + N天内是否被关闭）
    ↓
已被关闭 → 不展示，结束
    ↓
未被关闭 → 展示弹窗（轮播时只展示在时间段内的图片）
```

### 4.3 用户关闭流程

```
用户点击弹窗右上角 ×
    ↓
弹窗消失（写入本地：弹窗ID + 关闭时间戳）
    ↓
N天内再访问 → 不展示该弹窗
N天后记录失效 → 下次访问重新展示
```

## 5. 数据结构

### 5.1 弹窗配置表（popup_config）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| name | varchar(100) | 弹窗名称 |
| terminal_type | tinyint | 1=PC端 2=小程序端 |
| display_mode | tinyint | 1=单图 2=轮播 |
| activity_start_time | datetime | 活动整体开始时间 |
| activity_end_time | datetime | 活动整体结束时间 |
| image_url | varchar(500) | 单图模式图片OSS地址 |
| jump_type | varchar(50) | 单图模式跳转类型 |
| jump_params | varchar(500) | 单图模式跳转参数（JSON） |
| allow_close | tinyint | 0=不可关闭 1=可关闭 |
| close_expire_days | int | 关闭后再展示天数，默认7 |
| status | tinyint | 0=禁用 1=启用 |
| create_time | datetime | 创建时间 |
| update_time | datetime | 更新时间 |

## 6. 非功能性需求

| 类型 | 要求 |
|------|------|
| 性能 | 弹窗关闭响应 < 200ms；图片异步加载，不阻塞页面渲染 |
| 存储 | 本地记录有过期机制，自动清理超过N天的数据 |
| 兼容性 | PC端和微信小程序端各自的图片尺寸必须符合规范 |
| 稳定性 | 图片加载失败不阻塞页面；弹窗异常不影响主站运行 |

## 7. 验收标准

### 7.1 配置侧

- [ ] 运营可在平台端新增、编辑、删除、启用/禁用弹窗配置
- [ ] PC端上传图片尺寸必须为600×450px，否则阻止上传
- [ ] 小程序端上传图片尺寸必须为750×750px，否则阻止上传
- [ ] 图片展示时间超出活动整体时间时，系统提示并阻止保存
- [ ] 配置启用后立即生效，无缓存延迟

### 7.2 展示侧

- [ ] 单图模式：上传一张图片，在图片时间窗口内展示
- [ ] 轮播模式：上传2-6张图片，自动轮播在时间段内的图片
- [ ] 当前时间在活动整体时间外，弹窗不展示
- [ ] 当前时间无任何图片在展示时间段内，弹窗不展示（不写关闭记录）
- [ ] 多弹窗并存时，只展示最新创建的那一条
- [ ] 禁用状态的弹窗在任何情况下都不展示

### 7.3 交互侧

- [ ] 手动关闭开启时，右上角显示×，点击后弹窗消失，写入关闭记录
- [ ] 手动关闭关闭时，右上角不显示×，用户无法关闭
- [ ] 关闭后7天内再访问，弹窗不展示
- [ ] 7天后记录失效，弹窗重新展示
- [ ] PC端和小程序端关闭记录各自独立
- [ ] 轮播支持手动滑动切换
- [ ] 轮播仅1张图时不显示指示点，2张及以上显示
- [ ] 轮播图片加载失败时跳过该张，全部失败则不展示弹窗
- [ ] 跳转目标已删除时，点击无反应，不报错

### 7.4 性能

- [ ] 弹窗关闭操作响应 < 200ms
- [ ] 图片异步加载，不阻塞主站和小程序主页面渲染
- [ ] 本地存储记录有定期清理机制
