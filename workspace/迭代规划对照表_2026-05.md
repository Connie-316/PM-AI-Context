# [月] 迭代规划对照表

> 这个表是你自己维护的，不是TAPD。
> TAPD是团队的执行记录，这个表是你自己的规划对照表。
> 每月新建一个，文件名：`[月]迭代规划对照表.md`

---

## 目的

```
1. 对照TAPD迭代版本，看哪些被跳过了/删掉了
2. 记录哪些需求没经过你（组长直接写/改的）
3. 月底写版本总结时，知道全貌（不只是你处理的）
4. 早会汇报时用这个表，不报TAPD编号，只说任务名和状态
```

## 使用说明

```
每周一早上：
1. 打开TAPD，看当前迭代列表
2. 对照这个表，看哪些有变化
3. 更新"本周变化记录"
4. 有问题就在续期对话素材里找对应的话

每月25号：
1. 配合 monthly-plan 用
2. 把下月的规划填到"迭代版本详情"里
3. 标出关键任务（15号/31号）

月底写版本通知：
1. 从"红标任务"和"本月上线迭代"里摘
2. 通知发到大群前先检查一遍
```
