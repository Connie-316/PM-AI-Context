# 第一次怎么用（新手启动）

> 如果你是第一次用这个工作台，按这个步骤来。
> 只需要10分钟，就能开始你的第一个PRD。

---

## 第一步：先看看你有什么

```
打开 PM-AI-Context 文件夹，你会看到：

├── CLAUDE.md           ← AI的地图，你不用看
├── SKILL_GUIDE.md      ← 这个文件，你看这个
├── context/            ← 产品知识库（已有内容）
├── workspace/          ← 你的工作区
├── drafts/             ← 临时想法（暂时不用）
└── .claude/commands/   ← 所有Skill（AI用的，你不用看）
```

---

## 第二步：你的第一个PRD

### Step 1：告诉AI你要写什么

```
你说：快速PRD：帮我写一个[功能名称]的PRD

例子：
- "快速PRD：帮我写一个商品询价的PRD"
- "快速PRD：帮我写一个供应商入驻审核的PRD"
```

### Step 2：AI写完后你检查

```
- 看一遍，有没有遗漏
- 有就告诉AI："补充XX" 或 "XX要改一下"
- 没有就说"可以了"
```

### Step 3：AI自动做这些

```
你说"可以了"之后，AI会自动：
1. 保存版本到 history/
2. 审查文档（6个维度）
3. 推送到GitHub

你不需要做任何事。
```

---

## 第三步：给研发看

### PRD写完后

```
你说：输出研发文档：商品询价的PRD刚写完

AI会生成：
- 接口文档（研发可读）
- 研发需求确认单（让研发签字）
```

### 你把确认单打印/发给研发

```
研发说"明白了" → 签字 → 开始开发
```

---

## 第四步：每周一同步TAPD

```
打开TAPD → 看看这周有什么变化 → 说"TAPD对齐"

例子：
"TAPD对齐：组长删了v1466迭代，有个合同管理要加进去"
```

---

## 第五步：每月25号规划下月

```
说"月度规划" → AI问你这月和下月关键任务
你回答 → AI生成下月规划表（15号/31号上线）
```

---

## 你现在可以开始了

```
最简单的方式：

3. 等AI写完，检查，回复"可以了"

```

## 常见问题

### Q：我要打开哪个文件？
A：打开 PM-AI-Context 文件夹就行，你不需要打开任何.md文件。

### Q：Skill文件在哪里？
A：在 .claude/commands/ 里，你不需要进去看。

### Q：我的PRD存在哪里？
A：workspace/prd/ 里，每个PRD一个文件。

### Q：我想改之前写的PRD怎么办？
A：告诉AI："我要改[文件名]"，AI会帮你处理。

### Q：突然有个紧急需求怎么办？
A：直接说"快速PRD：紧急，帮我写一个XXX"，AI会标记优先级。
