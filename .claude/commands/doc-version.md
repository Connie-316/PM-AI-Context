# 文档版本管理 Skill（doc-version）

## 使用场景
当用户说"查版本"、"看历史"、"之前改了什么"、"回退"时调用。
也用于每次文档修改时自动记录版本。

---

## 核心原则

**每次文档修改必须：**
1. 不覆盖原文件，在 `history/` 下保留历史版本
2. 自动生成版本号（v1 / v2 / v3...）
3. 记录每次修改的变更说明
4. 每次修改后自动更新版本索引

---

## 版本命名规范

```
workspace/prd/
├── [功能名称]_[日期].md              ← 当前版本
└── history/
    ├── [功能名称]_[日期]_v1.md      ← 第1版
    ├── [功能名称]_[日期]_v2.md      ← 第2版
    └── ...

context/02_功能模块/
├── [功能名称].md                    ← 上线版本（最终版）
└── history/
    └── [功能名称]_v1.md             ← 历史归档版本
```

---

## 执行步骤

### 每次修改文档时的自动流程

```
Step 1: 修改文档前，先检查是否已有 history/ 目录
    ↓
Step 2: 如有 history/，将当前文件复制到 history/ 下（加当前版本号）
    ↓
Step 3: 创建新版本文件，覆盖当前文件
    ↓
Step 4: 更新版本索引文件 `[文件名]_versions.md`
    ↓
Step 5: 通知用户版本变化
```

### 版本索引文件格式

每个受控文档对应一个 `_[文件名]_versions.md`：

```markdown
# [功能名称] 版本记录

## 版本历史

| 版本 | 日期 | 修改说明 | 修改人 |
|------|------|---------|--------|
| v3 | 2026-05-21 | 补充时间规则、轮播模式 | [用户] |
| v2 | 2026-05-21 | 补充登录状态优先级 | [用户] |
| v1 | 2026-05-21 | 初始版本 | [用户] |

## 当前版本

- 文件：workspace/prd/[功能名称]_[日期].md
- 版本：v3
- 最后更新：2026-05-21
```

## 版本回退

### 查看历史

用户说"查看历史"时，显示：

```
[功能名称] 版本历史

v3 (2026-05-21) - 补充时间规则、轮播模式
v2 (2026-05-21) - 补充登录状态优先级
v1 (2026-05-21) - 初始版本

输入"回退到 v1"或"回退到 v2"可恢复指定版本
```

### 执行回退

```
用户说"回退到 v2"
    ↓
Step 1: 读取 history/[功能名称]_v2.md
    ↓
Step 2: 当前版本移到 history/（保留 v3）
    ↓
Step 3: v2 内容写入当前文件
    ↓
Step 4: 更新版本索引
    ↓
Step 5: 通知用户
```

---

## 自动触发

**以下情况自动记录版本，不需要用户说：**
1. 任何 Skill 修改了 PRD 文件后
2. 用户说"帮我改一下这个文档"后（修改前先保存当前版本）

---

## 不需要版本管理的文件

```
❌ drafts/ 里的草稿（临时文件，不值得版本管理）
❌ .claude/commands/ 下的 Skill 文件（Skill 有自己的更新逻辑）
❌ ui-demo-preview.html（可视化 Demo，不值得版本管理）
```

---

## 与其他 Skill 的配合

```
prd-write 修改 PRD
    ↓
doc-version 保存当前版本到 history/
    ↓
修改后 → doc-review 审查
    ↓
确认后 → git-push 推送（含 history/）
```

## 版本索引更新规则

每次新版本生成后，自动更新：

```markdown
## 版本历史（更新）

| 版本 | 日期 | 修改说明 | 修改人 |
|------|------|---------|--------|
| v4 | 2026-05-21 | 补充补充思考章节 | [用户] |
| v3 | 2026-05-21 | 补充时间规则、轮播模式 | [用户] |
| ... | ... | ... | ... |
```

## 精简版版本对比（每次自动生成）

每次保存版本时，自动生成一行变更摘要：

```markdown
## 变更摘要

**v3 → v4 (2026-05-21)**
- 修改：时间规则章节重写，删除了登录状态优先级
- 新增：轮播模式补充了 3 个边界条件
- 删除：冗余的"上下游耦合"章节

**v2 → v3 (2026-05-21)**
- 修改：主色从 #3370FF 改为 #1664FF
- 新增：登录状态优先级规则
```
