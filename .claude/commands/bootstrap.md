# Bootstrap 访谈模板

## 使用场景

当你需要系统性提取业务知识时（接手新产品、建立新 context 库、或者现有仓库太空洞需要重建），使用这个模板引导访谈。

---

## 核心原则

```
1. 用户的大脑是知识的源头 — 不要生成，要提取
2. 先问冷的，再问热的 — 公司背景/定位/术语先于 PRD/需求
3. 访谈比拍脑袋好 — 所有内容都应该来自用户亲口说的，而不是AI生成的
4. 问具体，不问抽象 — "这个词在你们业务里是什么意思"比"你们业务是做什么的"更好
```

---

## 访谈流程

### 第一轮：产品定位（Cold Content，最高 ROI）

```
问题1：你们这个产品/项目是做什么的？一句话概括。

问题2：谁会用？你们的目标客户是谁？有没有分不同角色（采购/财务/管理员）？

问题3：和市面上其他家比，你们有什么不同？客户为什么选你们不选别家？

问题4：你们的产品现在有多大？（用户数/SKU/订单量）
```

### 第二轮：行业知识（冷内容，需要专业词汇）

```
问题5：你们这个行业有什么特殊术语？比如你们这个领域的人常说的黑话？

问题6：你们的业务流程大概是什么样的？（从客户下单到收货的完整链路）

问题7：在这个行业里，新客户最容易遇到什么问题？
```

### 第三轮：团队与决策链（Warm Content）

```
问题8：你们的团队是怎么分工的？谁负责什么？

问题9：如果有个需求要做，流程是什么样的？（谁提→谁审→谁做→谁验）

问题10：你们现在的迭代节奏是怎样的？什么时候上线？
```

### 第四轮：现状与问题（Hot Content，最后问）

```
问题11：现在什么东西最让你头疼？

问题12：你们之前做过什么功能，但是后来没人用了？为什么？

问题13：你们认为这个产品最需要改进的地方是什么？
```

## 访谈输出模板

每次访谈完成后，生成以下结构（直接写入 context/ 对应目录）：

```markdown
# [产品/项目名] Bootstrap 报告

## 一、产品定位
[一句话定位]

## 二、目标用户
| 角色 | 说明 |
|------|------|

## 三、核心差异
[1-3句话描述差异化]

## 四、行业术语
| 术语 | 解释 |
|------|------|

## 五、业务流程
```
[流程图]
```

## 六、团队分工
| 角色 | 职责 |
|------|------|

## 七、当前最大问题
1. [问题1]
2. [问题2]
3. [问题3]
```

---

## 快速版本（已有基础时用）

如果时间有限，至少问这 3 个问题：

```
1. 你们的客户是谁？他们最常用的功能是什么？
2. 和竞品比，你们最大的优势是什么？
3. 现在什么东西让客户抱怨最多？
```

---

## 路由规则

访谈后，按以下规则路由：

| 内容类型 | 目标目录 |
|---------|---------|
| 产品定位/差异化/用户 | context/01_产品架构/产品简介.md |
| 行业术语 | context/01_产品架构/术语表.md |
| 业务流程/功能清单 | context/02_功能模块/ |
| 团队架构/接口人 | context/03_团队信息/团队架构.md |
| 技术决策/架构问题 | context/03_技术决策/ |

---

## 自动触发

```
以下情况自动建议 Bootstrap 访谈：
1. 用户说"这个产品我不熟"、"重新开始"
2. context/ 目录下超过 3 个文件为空
3. 新项目启动
4. 用户说"帮我系统性地整理一下知识库"
```
