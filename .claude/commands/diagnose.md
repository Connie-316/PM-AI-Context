# 知识库健康诊断 Skill（diagnose）

## 使用场景

当用户说"健康检查"、"诊断"、"仓库怎么样"、"缺什么"时调用。
每月至少执行一次，作为月度复盘的一部分。

---

## 核心原则

```
健康诊断不是为了批评，是为了找到杠杆点。
每次诊断后都要给出可执行的下一步行动，不是只给报告。
```

---

## 评估维度

| 维度 | 评估什么 |
|------|---------|
| **覆盖度** | 哪些知识领域有内容，哪些是空的 |
| **新鲜度** | 哪些文件很久没更新了（可能过时） |
| **深度** | 哪些领域只有一句话，没有详细内容 |
| **索引健康** | 索引文件是否和实际文件一致 |
| **生命周期** | 文档是否在正确阶段流动 |
| **冷内容优先度** | Cold 内容是否充足（最高杠杆） |

---

## 执行步骤

### Step 1：扫描仓库结构

```
对于每个目录，列出：
- 文件数量
- 最近更新的文件
- 空目录数量
- 新增但未索引的文件
```

### Step 2：评估六个维度

#### A. 覆盖度检查

对照标准知识领域，检查每个是否填充：

| 知识领域 | 应有文件 | 当前状态 |
|---------|---------|---------|
| 产品定位 | context/01_产品架构/产品简介.md | ✅/⚠️/❌ |
| 行业术语 | context/01_产品架构/术语表.md | ✅/⚠️/❌ |
| 竞品分析 | context/01_产品架构/竞品分析.md | ✅/⚠️/❌ |
| 团队架构 | context/03_团队信息/团队架构.md | ✅/⚠️/❌ |
| 接口人 | context/03_团队信息/接口人名单.md | ✅/⚠️/❌ |
| 已上线功能 | context/02_功能模块/已上线功能列表.md | ✅/⚠️/❌ |
| 技术决策 | context/03_技术决策/ | ✅/⚠️/❌ |
| 需求管理 | workspace/需求池管理.md | ✅/⚠️/❌ |
| PRD管理 | workspace/prd/ | ✅/⚠️/❌ |

#### B. 新鲜度检查

```
超过 60 天未更新的文件 → ⚠️ 可能过时
超过 90 天未更新的文件 → ❌ 需要审查
```

#### C. 深度检查

```
文件字数 < 50 字 → ⚠️ 太浅
文件字数 < 100 字 → 建议补充
```

#### D. 索引健康检查

```
索引文件中列的文件是否存在 → 缺失条目
实际存在的文件是否在索引中 → 孤儿文件
索引中的路径是否正确 → 死链接
```

#### E. 生命周期检查

```
drafts/ 有文件超过 7 天了吗？→ 推动晋升或删除
workspace/prd/ 有功能上线 2 周以上未归档吗？→ 推动归档
context/ 有文件 90 天以上未动吗？→ 审查是否还相关
```

#### F. 冷内容优先级检查

```
冷内容（产品定位/术语/团队）是否有实质性内容？
还是只有模板/空壳？
```

### Step 3：输出诊断报告

```markdown
# Context 健康报告
> 诊断时间：2026-05-21

## 一、覆盖度评分

| 知识领域 | 状态 | 说明 |
|---------|------|------|
| 产品定位 | ✅ | 有实质内容 |
| 行业术语 | ⚠️ | 有内容但深度不够 |
| 团队架构 | ⚠️ | 有框架但多个待补充项 |
| ... | ... | ... |

**覆盖度：X/9 领域填充**

## 二、新鲜度预警

| 文件 | 最后更新 | 天数 | 状态 |
|------|---------|------|------|
| [文件] | 2026-03-01 | 81天 | ⚠️ 过时 |

## 三、深度预警

| 文件 | 字数 | 状态 |
|------|------|------|
| [文件] | 45字 | ⚠️ 太浅 |

## 四、索引健康

| 目录 | 索引状态 |
|------|---------|
| context/01_产品架构/ | ✅ 同步 |
| context/02_功能模块/ | ⚠️ 漂移2处 |
| workspace/prd/ | ✅ 同步 |

## 五、生命周期状态

```
drafts/ 待处理：1个文件（超过7天）
workspace/prd/ 待归档：0个文件
context/ 待审查：1个文件（90天+未更新）
```

## 六、冷内容评估

```
✅ 产品简介 — 有实质内容
⚠️ 术语表 — 有基础词汇，但深度不够（缺少模具行业专业术语）
❌ 团队接口人 — 有框架但多个待补充项
```

2. 每月 25 号（月度规划前）
3. 每周五（周复盘前可选执行）
4. 新项目启动后第一次
```

## 与其他 Skill 的配合

```
diagnose 健康诊断
    ↓
发现覆盖度不足 → bootstrap 访谈补充
发现索引漂移 → maintain 同步
发现生命周期停滞 → evolve 推动晋升/归档
发现深度不够 → ingest 补充内容
```
