# 周复盘 Skill（weekly-review）

## 使用场景

当用户说"周复盘"、"本周总结"、"每周复盘"时调用。
每周五下班前执行。

---

## 核心原则

```
1. 复盘是为了发现问题，不是自我感动
2. 每个问题都要有后续行动，否则不记录
3. 复盘要客观，不美化，不推卸
4. 复盘结果要让相关人知道（团队/领导）
```

---

## 执行步骤

### Step 1：收集本周数据

```
从 TAPD 导出：
- 本周完成任务列表
- 本周新增需求列表
- 本周变更需求列表
- 关键任务进度

从 需求池管理.md 导出：
- 本周新进需求 [X] 个
- 本周完成需求 [X] 个
- 本周取消需求 [X] 个

从 PRD 状态追踪 导出：
- 进行中 PRD [X] 个
- 已上线功能 [X] 个
```

## 输出格式

```
1. 周复盘报告（Markdown）

```

---

## 触发时机

```
自动触发：每周五 17:00
手动触发：用户说"周复盘"、"本周总结"
```

---

## 与其他 Skill 的配合

```
weekly-review 生成复盘报告
    ↓
如有问题需要改流程 → 找组长沟通
    ↓
如有新需求 → 登记到 需求池管理.md
    ↓
如有PRD更新 → doc-version 保存版本
    ↓
确认后 → git-push
```
