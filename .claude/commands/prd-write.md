# 快速 PRD 撰写 Skill（prd-write）

## 使用场景
当用户说"帮我写个PRD"、"快速写一下"、"简单PRD"时调用。
**适用场景：单个功能点、轻量需求、不需要正式评审的文档。**

## 核心原则

- **快速直接** — 不要走需求分析流程，用户说什么就写什么
- **一段写完** — 不分两阶段，直接输出完整 PRD
- **三不原则** — 不写技术实现、不写设计参数、不写废话

## 执行步骤

直接根据用户需求，在 `workspace/prd/` 下创建 PRD 文件，命名格式：`prd/[功能名称]_[日期].md`

**创建后必须更新索引：**
1. 更新 `workspace/prd/CLAUDE.md`，在文件列表中新增这一行
2. 更新 `workspace/CLAUDE.md`，确认 prd/ 子目录已列出

## Context 加载规则

开始写之前，自动加载：
- `context/01_产品架构/产品简介.md`
- `context/02_功能模块/已上线功能列表.md`
- `context/01_产品架构/术语表.md`

确保与现有产品逻辑一致，不产生矛盾。

## 什么时候用这个 vs full-prd

| 场景 | 用哪个 |
|------|--------|
| 单个小功能、快速原型 | prd-write（这个） |
| 跨模块、团队评审、复杂需求 | full-prd（先走 requirements-analysis） |
| 用户只说一句话需求 | prd-write（这个） |
| 用户提供了需求分析文档 | full-prd |
